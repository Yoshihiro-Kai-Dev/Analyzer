# プロジェクト: wel_analyzer 開発計画書 (dataDiver 2.6.0 クローン)

## 1. 開発の目的と概要
本プロジェクトの目的は、既存の分析ツール「dataDiver (Ver 2.6.0)」のUXおよび分析ロジックをモデルとした、新規の勾配ブースティング決定木（GBDT）分析ツール「**wel-analyzer**」を開発することです。
ユーザーはノーコードでデータをアップロードし、テーブル間のリレーションを定義するだけで、高度な予測モデルの構築、要因分析、シミュレーションを行うことができます。

## 2. システム要件定義

### 2.1. ターゲットユーザー環境
- **ブラウザベース**: Chrome, Edge (Chromium版) での動作を前提とする。

### 2.2. 技術スタック（確定）
- **Frontend**: Next.js (TypeScript推奨)
- **Backend**: Python (FastAPI)
- **Database**: PostgreSQL (ローカルインストール版を使用)
- **ML Engine**: LightGBM または XGBoost
- **Data Processing**: Pandas, Polars
- **Infrastructure**: まずはローカル（スタンドアロン）で動作させ、将来的なDockerコンテナ化を見据えた構成（環境変数の分離など）とする。

## 4. 機能要件詳細（マニュアル網羅）

### Phase 1: データインジェスチョン（取り込み）機能
dataDiverの仕様に準拠し、以下のファイル処理を実装してください。

1.  **ファイルアップロード仕様**
    -   **形式**: CSV, ZIP（CSV 1ファイルのみ）
    -   **文字コード**: UTF-8 (BOMなし)
    -   **サイズ上限**: 300MB
    -   **自動命名**: ファイル名をデフォルトのデータ名称とする。

2.  **バリデーション**
    -   パスワード付きZIP、複数ファイルZIPは拒否。
    -   FastAPI側でストリーム処理を行い、メモリ効率を考慮する。

### Phase 2: データリレーション管理（関係図）機能
アップロードされた複数のテーブル（CSV）を結合するロジックを実装してください。

1.  **リレーション定義**
    -   **ID型項目**のみをキーとして結合可能。
    -   **カーディナリティ制御**: 「1:1」または「1:N」のみ許可。「N:M（多対多）」はエラーとして弾くロジックを実装すること。
    -   **ループ検知**: リレーション循環（ループ）を検知し警告する。
    -   **複合キー禁止**: 単一キーでのリレーションのみサポート。

2.  **可視化（マッチ率・ER図）**
    -   Next.js + React Flow等を用い、テーブル結合関係を視覚的に操作・表示するUIを構築。
    -   データ間の結合状況（マッチ率）を計算し可視化する。

### Phase 3: 分析設定（Analysis Configuration）
ユーザーが分析の「問い」を設定するウィザード形式のUIを作成してください。

1.  **基本設定フロー**
    -   **解析単位**: ID型カラムを選択。
    -   **解析対象（目的変数）**: 量的変数（売上等）または質的変数（フラグ等）。
    -   **解析課題**: 集計方法（合計/件数/平均）および方向性（高い/低い）の指定。

2.  **特徴量エンジニアリング**
    -   **フィルタリング**: 数値範囲、カテゴリ選択によるデータ絞り込み。
    -   **自動生成**: 日付からの「年月・曜日」抽出、ダミー変数化の自動処理。

### Phase 4: 分析実行・結果可視化エンジン
GBDTモデルを学習し、結果を解釈可能な形で提示してください。

1.  **可視化メトリクス**
    -   **順位 (Rank)**: 特徴量重要度に基づくランク付け。
    -   **クリアさ (Clarity)**: 要因の明確さを4段階分類。
    -   **影響度**: ポジティブ（青）/ネガティブ（赤）の色分け。
    -   **UI実装**: Recharts等を用いたインタラクティブなグラフ描画。

2.  **インタラクティブ分析**
    -   **除外・再計算**: 変数除外によるモデル再学習。
    -   **シミュレーション**: フロントエンド操作による予測値変動のリアルタイム試算。
    -   **数式出力**: モデルロジックの簡易数式表示。

### Phase 5: 予測（Prediction）と出力
学習済みモデルを新規データに適用する機能を実装してください。

1.  **予測実行**
    -   分析時と同様のリレーション構造を持つデータに対し予測を実行。

2.  **結果ダウンロード（CSV出力）**
    -   **ソート順**: 予測値（predicted_value）の降順。
    -   **必須カラム**: `unit_id`, `predicted_value`, `rank_small_to_large`, `rank_large_to_small`, `rank_percent`

## 5. 開発マイルストーン（Google Antigravity エージェント指示用）

以下の順序でコードを生成・実装してください。

1.  **Setup (Standalone)**:
    -   プロジェクトルートは `wel_analyzer` とする。
    -   Frontend (`wel_analyzer/frontend`): Next.jsセットアップ。
    -   Backend (`wel_analyzer/backend`): Python venv作成、FastAPIセットアップ。
    -   Database: ローカルPostgreSQLへの接続設定（`.env`ファイルによる環境変数管理を徹底し、将来のコンテナ化に備える）。
2.  **Data Module**:
    -   FastAPIでのCSVアップロードとDB保存処理。
    -   Pandasを用いた型判定ロジックの実装（日本語コメント必須）。
3.  **Relation Engine**:
    -   グラフ理論を用いたテーブル結合バリデーター実装。
    -   React Flowを用いたリレーション設定UI作成。
4.  **ML Core**:
    -   LightGBM学習パイプライン構築。
5.  **UI Implementation**:
    -   分析設定ウィザードおよび結果ダッシュボードの実装。
6.  **Prediction Export**:
    -   推論パイプラインとCSV出力機能の実装。
