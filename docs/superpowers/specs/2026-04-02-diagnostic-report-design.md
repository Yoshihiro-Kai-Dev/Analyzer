# モデル診断レポート設計

## 概要

ダッシュボードの「AI アナリティクス・インサイト」セクションを「モデル診断レポート」にリニューアルする。
現状のテンプレート的なメトリクス羅列を、実用的な診断・初学者向け解説・改善提案を含むレポートに置き換える。

## 背景・動機

- 現状の「AI アナリティクス・インサイト」は名前に反してLLMを使っておらず、テンプレートベースの静的テキスト生成
- 内容も評価指標・特徴量重要度セクションと重複しており、独自の価値が薄い
- ノーコードツールの初学者ユーザーが結果を解釈できるよう、診断と解説を充実させたい

## 方針

- LLMは使わず、ルールベースのロジックで診断を高度化する（コストゼロ・即時レスポンス）
- 各診断項目は「判定 → 根拠 → 初学者向け解説（引用ブロック） → 対策」の構造
- Markdownテキストとして生成し、ReactMarkdownでレンダリング（カード形式UIは採用しない）

## 変更対象ファイル

### 1. `backend/app/services/ml_service.py`

#### trainメトリクスの追加計算

過学習検出のため、testデータに加えてtrainデータへの予測メトリクスも計算する。

- メトリクス計算部分（199行目付近）で `y_train_pred = sk_model.predict(X_train_s)` を追加
- `train_metrics` dictを算出（train R2/Accuracy等）
- `_generate_diagnostic_report` に `metrics`（test）と `train_metrics`（train）の両方を渡す

#### `_generate_simple_insight` → `_generate_diagnostic_report` にリライト

**入力パラメータ**:
- `task_type: str` — "regression" or "classification"
- `metrics: dict` — テストデータのメトリクス
- `train_metrics: dict` — 学習データのメトリクス（過学習検出用）
- `fi_list: list` — 特徴量重要度リスト
- `model_type: str` — "gradient_boosting" or "logistic_regression"

**診断項目と判定ロジック**:

| # | 項目 | 判定基準 |
|---|---|---|
| 1 | モデル精度の総合判定 | 回帰: R2 > 0.8 良好 / 0.5〜0.8 一定 / < 0.5 改善要。分類: Accuracy > 0.9 / 0.7 / それ以下 |
| 2 | 過学習の兆候検出 | train と test のメトリクス差が 0.1 以上で警告、0.2 以上で要改善 |
| 3 | クラス不均衡の指摘 | 分類のみ。Precision と Recall の差が 0.15 以上で警告 |
| 4 | 特徴量の支配度チェック | Top1 の重要度が全体の 50% 以上で警告 |
| 5 | 低寄与特徴量の指摘 | 重要度 < 0.01 の特徴量をリストアップ |
| 6 | 改善のヒント | 上記 1〜5 の診断結果に基づいて条件分岐で生成 |

**出力**: Markdownテキスト（`ai_analysis_text` カラムにそのまま格納）

### 2. `frontend/src/app/projects/[projectId]/dashboard/page.tsx`

- セクションタイトル: 「AI アナリティクス・インサイト」→「モデル診断レポート」
- アイコン: `Sparkle` → 診断らしいアイコンに変更
- HTML `<details><summary>` でアコーディオン化（デフォルト閉じ）
- ReactMarkdownレンダリングはそのまま維持

## 変更しないもの

- DBスキーマ（`ai_analysis_text` カラムをそのまま使用）
- APIレスポンス形式
- 他のダッシュボードセクション（評価指標・特徴量重要度等）

## 出力例

```markdown
## モデル診断レポート（回帰 / 勾配ブースティング）

### モデル精度
良好な予測精度です（R² = 0.82, RMSE = 12.34）。
> R²は「モデルがデータの変動をどれだけ説明できるか」を表す指標で、
> 1.0に近いほど優秀です。0.8以上は実用的な水準といえます。

### 過学習チェック
⚠ 過学習の兆候があります（学習データ R²: 0.95 → テストデータ R²: 0.82）
> 学習データでは高精度なのに、未知のデータでは精度が落ちている状態です。
> モデルがデータの「ノイズ」まで覚えてしまっている可能性があります。
>
> **対策**: 特徴量を減らす、またはデータ件数を増やすことを検討してください。

### クラス不均衡チェック
（回帰タスクのため省略）

### 特徴量の支配度
⚠ 特徴量「年収」が重要度の 62% を占めています。
> 1つの特徴量にモデルが頼りすぎている状態です。
> その特徴量に欠損やノイズがあると、予測全体に大きく影響します。
>
> **対策**: 「年収」に関連する別の特徴量を追加するか、
> この特徴量が本当に予測に使って良いものか確認してください。

### 低寄与の特徴量
以下の特徴量はモデルにほとんど影響を与えていません:
- 顧客ID（重要度: 0.00）
- 登録日_月（重要度: 0.01）
> これらを除外しても精度はほぼ変わりません。
> 特徴量を減らすことでモデルがシンプルになり、過学習の防止にもつながります。

### 💡 改善のヒント
1. 過学習の兆候があるため、低寄与の特徴量（顧客ID、登録日_月）を除外して再学習してみてください。
2. 「年収」への依存度が高いため、年収を補完する特徴量（勤続年数、職種など）があれば追加を検討してください。
```

**表示ルール**:
- 各セクションは「判定 → 根拠 → 解説（引用ブロック） → 対策」の流れ
- 該当しない診断項目は省略（例: 回帰タスクでクラス不均衡チェック）
- 問題がない項目は「✅ 問題なし」と短く表示
- 最後の「改善のヒント」は上の診断結果から自動で組み立てる
